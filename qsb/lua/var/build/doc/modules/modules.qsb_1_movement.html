Module modules.qsb_1_movement

Event

QSB.ScriptEvents
Events, auf die reagiert werden kann.
  • EntityArrived Ein Entity hat das Ziel erreicht. (Parameter: EntityID, Position, DataIndex)
  • EntityStuck Ein Entity kann das Ziel nicht erreichen. (Parameter: EntityID, Position, DataIndex)
  • EntityAtCheckpoint Ein Entity hat einen Wegpunkt erreicht. (Parameter: EntityID, Position, DataIndex)
  • PathFindingFinished Ein Pfad wurde erfolgreich gefunden (Parameter: PathID)
  • PathFindingFailed Ein Pfad konnte nicht ermittelt werden (Parameter: PathID)

Bewegung

API.MoveEntity (_Entity, _Target, _IgnoreBlocking)
Bewegt ein Entity zum Zielpunkt.

Wenn das Ziel zu irgend einem Zeitpunkt nicht erreicht werden kann, wird die Bewegung abgebrochen und das Event QSB.ScriptEvents.EntityStuck geworfen.

Das Ziel gilt als erreicht, sobald sich das Entity nicht mehr bewegt. Dann wird das Event QSB.ScriptEvents.EntityArrived geworfen.

Parameters:

  • _Entity Bewegtes Entity (Skriptname oder ID)
  • _Target Ziel (Skriptname, ID oder Position)
  • _IgnoreBlocking boolean Direkten Weg benutzen

Beispiel:

  • -- Beispiel #1: Normale Bewegung
API.MoveEntity("Marcus", "Target");
  • -- Beispiel #2: Schiff bewegen
API.MoveEntity("Ship", "Harbor", true);

API.MoveEntityOnCheckpoints (_Entity, _WaypointList, _IgnoreBlocking)
Bewegt ein Entity über den angegebenen Pfad.

Wenn das Ziel zu irgend einem Zeitpunkt nicht erreicht werden kann, wird die Bewegung abgebrochen und das Event QSB.ScriptEvents.EntityStuck geworfen.

Jedes Mal wenn das Entity einen Wegpunkt erreicht hat, wird das Event QSB.ScriptEvents.EntityAtCheckpoint geworfen.

Das Ziel gilt als erreicht, sobald sich das Entity nicht mehr bewegt. Dann wird das Event QSB.ScriptEvents.EntityArrived geworfen.

Parameters:

  • _Entity Bewegtes Entity (Skriptname oder ID)
  • _WaypointList Liste mit Wegpunkten
  • _IgnoreBlocking boolean Direkten Weg benutzen

Beispiel:

    API.MoveEntityOnCheckpoints("Marcus", {"WP1", "WP2", "WP3", "Target"});

API.StartPathfinding (_StartPosition, _EndPosition, _NodeFilter)
Beginnt die Wegsuche zwischen zwei Punkten.

Der Pfad wird nicht sofort zurückgegeben. Stattdessen eine ID. Der Pfad wird asynchron gesucht, damit das Spiel nicht einfriert. Wenn die Pfadsuche abgeschlossen wird, werden entsprechende Events ausgelöst.

  • QSB.ScriptEvents.PathFindingFinished
    Ein Pfad zwischen den Punkten wurde gefunden.
  • QSB.ScriptEvents.PathFindingFailed
    Es konnte kein Pfad gefunden werden.

Wird der Node Filter weggelassen, wird automatisch eine Funktion erstellt, die alle Positionen ausschließt, die geblockt sind.

Parameters:

  • _StartPosition Beginn des Pfad (Position, Skriptname oder ID)
  • _EndPosition Ende des Pfad (Position, Skriptname oder ID)
  • _NodeFilter function (Optional) Filterfunktion für Wegpunkte

Beispiel:

  • -- Beispiel #1: Standard Wegsuche
MyPathID = API.StartPathfinding("Start", "End");
  • -- Beispiel #2: Wegsuche mit Filter
MyPathID = API.StartPathfinding("Start", "End", function(_CurrentNode, _AdjacentNodes)
    -- Position verwerfen, wenn sie im Blocking ist
    if Logic.DEBUG_GetSectorAtPosition(_CurrentNode.X, _CurrentNode.Y) == 0 then
        return false;
    end
    -- Position verwerfen, wenn sie auf Territorium 16 liegt
    if Logic.GetTerritoryAtPosition(_CurrentNode.X, _CurrentNode.Y) == 16 then
        return false;
    end
    -- Position akzeptieren
    return true;
end);
