Module (4) Auftragsnotizen

Stellt Behavior bereit um die Tagebuchfunktion über den Assistenten nutzbar zu machen.

Event

QSB.ScriptEvents Events, auf die reagiert werden kann.

Reward

Reward_JournaHighlight (_QuestName, _EntryName, _Highlighted) Hebt einen Eintrag im Tagebuch hervor oder setzt ihn auf normal zurück.
Reward_JournalEnable (_QuestName, _Active) Zeigt das Tagebuch für einen Quest an oder versteckt es.
Reward_JournalRemove (_QuestName, _EntryName) Entfernt einen Tagebucheintrag von einem Quest.
Reward_JournalWrite (_QuestName, _EntryName, _EntryText) Schreibt einen Tagebucheintrag zu dem angegebenen Quest.

Anwenderfunktionen

API.AddJournalEntryToQuest (_ID, _Quest)
Fügt einen Tagebucheintrag zu einem Quest hinzu.

Parameters:

  • _ID number ID des Eintrag
  • _Quest boolean Name des Quest

Beispiel:

    API.AddJournalEntryToQuest(_ID, _Quest);
API.AllowNotesForQuest (_Quest, _Flag)
Aktiviert die Möglichkeit, selbst Notizen zu schreiben.

Hinweis: Die Zusatzinformationen müssen für den Quest aktiv sein.

Parameters:

  • _Quest string Name des Quest
  • _Flag boolean Notizen aktivieren

Beispiel:

    -- Deaktivieren
API.AllowNotesForQuest("MyQuest", false);
-- Aktivieren
API.AllowNotesForQuest("MyQuest", true);
API.AlterJournalEntry (_ID, _Text)
Ändert den Text einer Zusatzinformation.

Hinweis: Der neue Text bezieht sich auf den Eintrag mit der ID. Ist der Eintrag für alle Quests sichtbar, wird er in allen Quests geändert. Kopien eines Eintrags werden nicht berücksichtigt.

Hinweis: Formatierungsbefehle sind deaktiviert.

Parameters:

  • _ID number ID des Eintrag
  • _Text Neuer Text

Beispiel:

    API.AlterJournalEntry(SomeEntryID, "Das ist der neue Text.");
API.CreateJournalEntry (_Text)
Fugt eine Zusatzinformation für diesen Quests hinzu.

Hinweis: Die erzeugte ID ist immer eindeutig für alle Einträge, ungeachtet ob sie einem Quest zugeordnet sind oder nicht.

Hinweis: Der Questname kann durch nil ersetzt werden. In diesem Fall erscheint der Eintrag bei allen Quests (für die das Feature aktiviert ist). Und das so lange, bis er wieder gelöscht wird.

Hinweis: Formatierungsbefehle sind deaktiviert.

Parameters:

  • _Text string Text der Zusatzinfo

Rückgabe:

    number ID des neuen Eintrags

Beispiel:

    local NewEntryID = API.CreateJournalEntry("Wichtige Information zum Anzeigen");
API.DeleteJournalEntry (_ID)
Entfernt einen Eintrag aus den Zusatzinformationen.

Hinweis: Ein Eintrag wird niemals wirklich gelöscht, sondern nur unsichtbar geschaltet.

Parameters:

  • _ID number ID des Eintrag

Beispiel:

    API.DeleteJournalEntry(SomeEntryID);
API.HighlightJournalEntry (_ID, _Important)
Hebt einen Eintrag aus den Zusatzinformationen als wichtig hervor oder setzt ihn zurück.

Hinweis: Wichtige Einträge erscheinen immer als erstes und sind durch rote Färbung hervorgehoben. Eigene Farben in einer Nachricht beeinträchtigen die rote hervorhebung.

Parameters:

  • _ID number ID des Eintrag
  • _Important boolean Wichtig Markierung

Beispiel:

    API.HighlightJournalEntry(SomeEntryID, true);
API.RemoveJournalEntryFromQuest (_ID, _Quest)
Entfernt einen Tagebucheintrag von einem Quest.

Parameters:

  • _ID number ID des Eintrag
  • _Quest boolean Name des Quest

Beispiel:

    API.RemoveJournalEntryFromQuest(_ID, _Quest);
API.RestoreJournalEntry (_ID)
Stellt einen gelöschten Eintrag in den Zusatzinformationen wieder her.

Parameters:

  • _ID number ID des Eintrag

Beispiel:

    API.RestoreJournalEntry(SomeEntryID);
API.ShowJournalForQuest (_Quest, _Flag)
Aktiviert oder Deaktiviert die Verfügbarkeit der Zusatzinformationen für den übergebenen Quest.

Hinweis: Die Sichtbarkeit der Zusatzinformationen für einzelne Quests ist generell deaktiviert und muss explizit aktiviert werden.

Hinweis: Der Button wird auch dann angezeigt, wenn es noch keine Zusatzinformationen für den Quest gibt.

Parameters:

  • _Quest string Name des Quest
  • _Flag boolean Zusatzinfos aktivieren

Beispiel:

    -- Deaktivieren
API.ShowJournalForQuest("MyQuest", false);
-- Aktivieren
API.ShowJournalForQuest("MyQuest", true);
