Module (2) Interaktive Objekte

Der Spieler muss bis zu 4 interaktive Objekte benutzen.

Functions

API.DisposeObject (_ScriptName) Zerstört die Interation mit dem Objekt.

Anwenderfunktionen

API.InteractiveObjectActivate (_ScriptName, _State, ...) Aktiviert ein Interaktives Objekt, sodass es von den Spielern aktiviert werden kann.
API.InteractiveObjectDeactivate (_ScriptName, ...) Deaktiviert ein interaktives Objekt, sodass es nicht mehr von den Spielern benutzt werden kann.
API.InteractiveObjectSetQuestName (_Key, _Text) Erzeugt eine Beschriftung für Custom Objects.
API.ResetObject (_ScriptName) Setzt das interaktive Objekt zurück.
API.SetupObject (_Description) Erzeugt ein einfaches interaktives Objekt.

Event

QSB.ScriptEvents Events, auf die reagiert werden kann.


Functions

API.DisposeObject (_ScriptName)
Zerstört die Interation mit dem Objekt.

Hinweis: Das Entity selbst wird nicht zerstört.

Parameters:

  • _ScriptName string Skriptname des Objektes

Verwandte Themen:

  • API.SetupObject
  • API.ResetObject

Beispiel:

    API.DisposeObject("MyObject");

Anwenderfunktionen

API.InteractiveObjectActivate (_ScriptName, _State, ...)
Aktiviert ein Interaktives Objekt, sodass es von den Spielern aktiviert werden kann.

Optional kann das Objekt nur für einen bestimmten Spieler aktiviert werden.

Der State bestimmt, ob es immer aktiviert werden kann, oder ob der Spieler einen Helden benutzen muss. Wird der Parameter weggelassen, muss immer ein Held das Objekt aktivieren.

Parameters:

  • _ScriptName string Skriptname des Objektes
  • _State number State des Objektes
  • ... number (Optional) Liste mit PlayerIDs
API.InteractiveObjectDeactivate (_ScriptName, ...)
Deaktiviert ein interaktives Objekt, sodass es nicht mehr von den Spielern benutzt werden kann.

Optional kann das Objekt nur für einen bestimmten Spieler deaktiviert werden.

Parameters:

  • _ScriptName string Scriptname des Objektes
  • ... number (Optional) Liste mit PlayerIDs
API.InteractiveObjectSetQuestName (_Key, _Text)
Erzeugt eine Beschriftung für Custom Objects.

Im Questfenster werden die Namen von Custom Objects als ungesetzt angezeigt. Mit dieser Funktion kann ein Name angelegt werden.

Parameters:

  • _Key string Typname des Entity
  • _Text Text der Beschriftung

Beispiel:

  • -- Beispiel #1: Einfache Beschriftung
API.InteractiveObjectSetQuestName("D_X_ChestOpenEmpty", "Leere Schatztruhe");
  • -- Beispiel #1: Multilinguale Beschriftung
API.InteractiveObjectSetQuestName("D_X_ChestClosed", {de = "Schatztruhe", en = "Treasure"});
API.ResetObject (_ScriptName)
Setzt das interaktive Objekt zurück. Dadurch verhält es sich, wie vor der Aktivierung durch den Spieler.

Hinweis: Das Objekt muss wieder per Skript aktiviert werden, damit es im Spiel ausgelöst werden.

Parameters:

  • _ScriptName string Skriptname des Objektes

Verwandte Themen:

  • API.SetupObject
  • API.InteractiveObjectActivate
  • API.InteractiveObjectDeactivate

Beispiel:

    API.ResetObject("MyObject");
API.SetupObject (_Description)
Erzeugt ein einfaches interaktives Objekt.

Dabei können alle Entities als interaktive Objekte behandelt werden, nicht nur die, die eigentlich dafür vorgesehen sind.

Die Parameter des interaktiven Objektes werden durch seine Beschreibung festgelegt. Die Beschreibung ist eine Table, die bestimmte Werte für das Objekt beinhaltet. Dabei müssen nicht immer alle Werte angegeben werden.

Achtung: Wird eine Straße über einem Objekt platziert, während die Kosten bereits bezahlt und auf dem Weg sind, läuft die Aktivierung ins Leere. Zwar wird das Objekt zurückgesetzt, doch die bereits geschickten Waren sind dann futsch.

Mögliche Angaben:

Feldname Typ Beschreibung Optional
Name string Der Skriptname des Entity, das zum interaktiven Objekt wird. nein
Texture table Angezeigtes Icon des Buttons. Die Icons können auf die Icons des Spiels oder auf eigene Icons zugreifen.
- Spiel-Icons: {x, y, Spielversion}
- Benutzerdefinierte Icons: {x, y, Dateinamenpräfix}		 ja
Title string Angezeigter Titel des Objekt ja
Text string Angezeigte Beschreibung des Objekt ja
Distance number Die minimale Entfernung zum Objekt, die ein Held benötigt um das objekt zu aktivieren. ja
Player number|table Spieler, der/die das Objekt aktivieren kann/können. ja
Waittime number Die Zeit, die ein Held benötigt, um das Objekt zu aktivieren. ja
Replacement number Entity, mit der das Objekt nach Aktivierung ersetzt wird. ja
Costs Eine Table mit dem Typ und der Menge der Kosten. (Format: {Typ, Menge, Typ, Menge}) ja
Reward table Der Warentyp und die Menge der gefundenen Waren im Objekt. (Format: {Typ, Menge}) ja
State number Bestimmt, wie sich der Button des interaktiven Objektes verhält. ja
Condition function Eine zusätzliche Aktivierungsbedinung als Funktion. ja
ConditionInfo string Nachricht, die angezeigt wird, wenn die Bedinung nicht erfüllt ist. ja
Action function Eine Funktion, die nach der Aktivierung aufgerufen wird. ja
RewardResourceCartType number Erlaubt, einen anderern Karren für Rohstoffkosten einstellen. ja
RewardGoldCartType number Erlaubt, einen anderern Karren für Goldkosten einstellen. ja
CostResourceCartType number Erlaubt, einen anderern Karren für Rohstoffbelohnungen einstellen. ja
CostGoldCartType number Erlaubt, einen anderern Karren für Goldbelohnung einstellen. ja

Parameters:

  • _Description table Beschreibung

Verwandte Themen:

  • API.ResetObject
  • API.InteractiveObjectActivate
  • API.InteractiveObjectDeactivate

Beispiel:

    API.SetupObject {
    Name     = "hut",
    Distance = 1500,
    Reward   = {Goods.G_Gold, 1000},
};

Event

QSB.ScriptEvents
Events, auf die reagiert werden kann.
  • ObjectClicked Der Spieler klickt auf den Button des IO (Parameter: ScriptName, KnightID, PlayerID)
  • ObjectInteraction Es wird mit einem interaktiven Objekt interagiert (Parameter: ScriptName, KnightID, PlayerID)
  • ObjectDelete Eine Interaktion wird von einem Objekt entfernt (Parameter: ScriptName)
  • ObjectReset Der Zustand eines interaktiven Objekt wird zurückgesetzt (Parameter: ScriptName)
