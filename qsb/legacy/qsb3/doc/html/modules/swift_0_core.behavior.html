    Goal_BuildWall (_PlayerID, _Position1, _Position2)
    Eine Mauer muss gebaut werden um die Bewegung eines Spielers einzuschränken. Einschränken bedeutet, dass sich der angegebene Spieler nicht von Punkt A nach Punkt B bewegen kann, weil eine Mauer im Weg ist. Die Punkte sind frei wählbar. In den meisten Fällen reicht es, Marktplätze anzugeben.

    Beispiel: Spieler 3 ist der Feind von Spieler 1, aber Bekannt mit Spieler 2. Wenn er sich nicht mehr zwischen den Marktplätzen von Spieler 1 und 2 bewegen kann, weil eine Mauer dazwischen ist, ist das Ziel erreicht.

    Achtung: Bei Monsun kann dieses Ziel fälschlicher Weise als erfüllt gewertet werden, wenn der Weg durch Wasser blockiert wird!

    Parameters:

    • _PlayerID PlayerID, die blockiert wird
    • _Position1 Erste Position
    • _Position2 Zweite Position

    Goal_CustomVariables (_Name, _Relation, _Value)
    Eine benutzerdefinierte Variable muss einen bestimmten Wert haben.

    Custom Variables können ausschließlich Zahlen enthalten. Bevor eine Variable in einem Goal abgefragt werden kann, muss sie zuvor mit Reprisal_CustomVariables oder Reward_CutsomVariables initialisiert worden sein.

    Vergleichsoperatoren

    • == - Werte müssen gleich sein
    • ~= - Werte müssen ungleich sein
    • > - Variablenwert größer Vergleichswert
    • >= - Variablenwert größer oder gleich Vergleichswert
    • < - Variablenwert kleiner Vergleichswert
    • <= - Variablenwert kleiner oder gleich Vergleichswert

    Parameters:

    • _Name Name der Variable
    • _Relation Vergleichsoperator
    • _Value Wert oder andere Custom Variable mit wert.
    Goal_Deliver (_GoodType, _GoodAmount, _OtherTarget, _IgnoreCapture)
    Einem Spieler müssen Rohstoffe oder Waren gesendet werden.

    In der Regel wird zum Auftraggeber gesendet. Es ist aber möglich auch zu einem anderen Zielspieler schicken zu lassen. Wird ein Wagen gefangen genommen, dann muss erneut geschickt werden. Optional kann dem Spieler auch erlaubt werden, den Karren zurückzuerobern.

    Parameters:

    • _GoodAmount Menga der Ware
    • _OtherTarget Anderes Ziel als Auftraggeber
    • _IgnoreCapture Wagen kann zurückerobert werden
    Goal_DestroyAllPlayerUnits (_PlayerID)
    Alle Einheiten des Spielers müssen zerstört werden.

    Achtung: Bei normalen Parteien, welche ein Dorf oder eine Stadt besitzen, ist Goal_DestroyPlayer besser geeignet!

    Parameters:

    • _PlayerID ID des Spielers
    Goal_DestroyPlayer (_PlayerID)
    Eine andere Partei muss besiegt werden.

    Die Partei gilt als besiegt, wenn ein Hauptgebäude (Burg, Kirche, Lager) zerstört wurde. Achtung: Bei Banditen ist dieses Behavior wenig sinnvoll, da sie nicht durch zerstörung ihres Hauptzeltes vernichtet werden. Hier bietet sich Goal_DestroyAllPlayerUnits an.

    Parameters:

    • _PlayerID ID des Spielers
    Goal_DestroyScriptEntity (_ScriptName)
    Ein benanntes Entity muss zerstört werden.

    Ein Entity gilt als zerstört, wenn es nicht mehr existiert oder während der Laufzeit des Quests seine Entity-ID oder den Besitzer verändert.

    Achtung: Helden können nicht direkt zerstört werden. Bei ihnen genügt es, wenn sie sich "in die Burg zurückziehen".

    Parameters:

    • _ScriptName Skriptname des Ziels
    Goal_DestroyType (_EntityType, _Amount, _PlayerID)

    Goal_Diplomacy (_PlayerID, _Relation, _State)
    Es muss ein bestimmter Diplomatiestatus zu einer anderen Patei erreicht werden. Der Status kann eine Verbesserung oder eine Verschlechterung zum aktuellen Status sein.

    Die Relation kann entweder auf kleiner oder gleich (<=), größer oder gleich (>=), oder exakte Gleichheit (==) eingestellt werden. Exakte GLeichheit ist wegen der Gefahr eines Soft Locks mit Vorsicht zu genießen.

    Parameters:

    • _Relation Größer-Kleiner-Relation
    • _State Diplomatiestatus

    Goal_MapScriptFunction (_FunctionName)
    Führt eine Funktion im Skript als Goal aus.

    Die Funktion muss entweder true, false oder nichts zurückgeben.

    • true: Erfolgreich abgeschlossen
    • false: Fehlschlag
    • nichts: Zustand unbestimmt

    Anstelle eines Strings kann beim Einsatz im Skript eine Funktionsreferenz übergeben werden. In diesem Fall werden alle weiteren Parameter direkt an die Funktion weitergereicht.

    Parameters:
