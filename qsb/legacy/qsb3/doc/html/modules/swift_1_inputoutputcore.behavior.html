Module swift_1_inputoutputcore.behavior

Fügt Behavior zur Aufforderung einer Eingabe hinzu.

Goal

Goal_Decide (_Text, _Title, _Labels)
Lässt den Spieler zwischen zwei Antworten wählen.

Dabei kann zwischen den Labels Ja/Nein und Ok/Abbrechen gewählt werden.

Hinweis: Es können nur geschlossene Fragen gestellt werden. Dialoge müssen also immer mit Ja oder Nein beantwortbar sein oder auf Okay und Abbrechen passen.

Multiplayer
Nicht für Multiplayer geeignet.

Parameters:

  • _Text Fenstertext
  • _Title Fenstertitel
  • _Labels Label der Buttons
Goal_InputDialog (_Passwords, _Trials, _Message)
Der Spieler muss im Chatdialog eine Eingabe tätigen.

Das Behaviour kann auch eingesetzt werden, um ein Passwort zu prüfen. In diesem Fall wird die Eingabe mit dem Passwort verglichen. Die Anzal der Versuche bestimmt, wie oft falsch eingegeben werden darf.

Wenn die Anzahl der Versuche begrenzt ist, wird eine Srandardnachricht mit den übrigen Versuchen angezeigt. Optional kann eine Nachricht angegeben werden, die stattdessen nach jeder Falscheingabe, außer der letzten, angezeigt wird.

Achtung: Alle aktiven Quests mit einem Input Behavior werden die erste Eingabe annehmen, die getätigt wird. Dabei ist es egal, ob der Input durch sie selbst oder extern aktiviert wurde.

Multiplayer
Nicht für Multiplayer geeignet.

Parameters:

  • _Passwords Liste der Passwörter
  • _Trials Anzahl versuche (0 für unendlich)
  • _Message Alternative Fehlernachricht
