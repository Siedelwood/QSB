Module swift_2_entitysearch.api

Stellt eine bessere Suche nach Entities bereit.

Die Suche nach Entities, speziell solcher ohne eine Kategorie oder eines Spielers, gestaltet sich oft schwer. Dieses Mudul wartet mit einer neuen Suchmethode auf, die garantiert alle Entities findet und keine Grenzen kennt.

Die Suche nach Entities wird über Prädikate gesteuert. Ein Prädikat ist ein Kriterium, dass das Ergebnis der Entity-Suche anhand von Paramern enschränkt. Dabei gibt es Prädikate, welche schneller als andere abgearbeitet werden. Die Reihenfolge, in der sie gelistet werden, ist also wichtig.

Vorausgesetzte Module:

  • (1) Jobs Core

Fields

QSB.SearchPredicate
Mögliche Prädikate für die Suche.
  • OfID (_ID, ...) - Schränkt auf eine bestimmte EntityID ein.
  • OfPlayer (_Player, ...) - Schränkt auf Entities eines bestimmten Spielers ein.
  • OfName (_Name, ...) - Schränkt auf eine bestimmten Skriptnamen ein.
  • OfNamePrefix (_Prefix, ...) - Schränkt auf Entities ein, deren Name mit dem Präfix beginnt.
  • OfNameSuffix (_Suffix, ...) - Schränkt auf Entities ein, deren Name mit dem Suffix endet.
  • OfType (_Type, ...) - Schränkt auf Entities mit dem Typen ein.
  • OfCategory (_Category, ...) - Schränkt auf Entities mit der Kategorie ein.
  • InArea (_X, _Y, _AreaSize) - Schränkt auf Entities im Gebiet ein.
  • InTerritory (_Territory, ...) - Schränkt auf Entities im Territorium.
  • IsBuilding () - Schränkt auf Gebäude ein.
  • IsFinishedBuilding () - Schränkt auf fertig gebaute Gebäude ein.
  • IsSettler () - Schränkt auf Siedler ein.

    Predikate können verknüpft werden über Operatoren.

    • NOT (_Predicate) - Negiert das Ergebnis des Prädikat.
    • ALL (...) - Alle Prädikate müssen wahr sein.
    • ANY (...) - Mindestes ein Prädikat muss wahr sein
    • XOR (...) - Exklusiv 1 aus allen Predikaten muss wahr sein.

Verwandte Themen:

    API.CommenceEntitySearch

API.CommenceEntitySearch (...)
Führt eine benutzerdefinierte Suche nach Entities aus.

Achtung: Die Reihenfolge der angewandten Predikate hat maßgeblichen Einfluss auf die Dauer der Suche. Während Abfragen auf den Besitzer oder den Typ schnell gehen, dauern Gebietssuchen lange! Es ist daher klug, zuerst Kriterien auszuschließen, die schnell bestimmt werden!

Multiplayer
Im Multiplayer kann diese Funktion nur in synchron ausgeführtem Code benutzt werden, da es sonst zu Desyncs komm.

Parameters:

  • ... table Liste mit Suchprädikaten

Returns:

    table Liste mit Ergebnissen

Verwandte Themen:

    QSB.SearchPredicate

Beispiel:

    -- Es werden alle Kühe und Schafe von Spieler 1 gefunden, die nicht auf den
-- Territorien 7 und 15 sind.
local Result = API.CommenceEntitySearch(
    -- Nur Entities von Spieler 1 akzeptieren
    {QSB.SearchPredicate.OfPlayer, 1},
    -- Nur Entities akzeptieren, die Kühe oder Schafe sind.
    {ANY,
     {QSB.SearchPredicate.OfCategory, EntityCategories.SheepPasture},
     {QSB.SearchPredicate.OfCategory, EntityCategories.CattlePasture}},
    -- Nur Entities akzeptieren, die nicht auf den Territorien 7 und 15 sind.
    {ALL,
     {NOT, {QSB.SearchPredicate.InTerritory, 15}},
     {NOT, {QSB.SearchPredicate.InTerritory, 7}}}
);
