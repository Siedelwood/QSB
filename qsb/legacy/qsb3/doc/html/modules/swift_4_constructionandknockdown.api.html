Module swift_4_constructionandknockdown.api

Ermöglicht Abriss und Bau für den Spieler einzuschränken.

Hinweis: Jegliche Enschränkungen funktionieren nur für menschlische Spieler. Die KI wird sie alle ignorieren!

Eine Baubeschränkung oder ein Abrissschutz geben eine ID zurück, über die seibiger dann gelöscht werden kann.

Es gibt zudem eine Hierarchie, nach der die einzelnen Checks durchgeführt werden. Dabei wird nach Art des betroffenen Bereiches und nach Art des betroffenen Subjektes unterschieden.

Nach Art des Bereiches:

  1. Custom-Funktionen
  2. Durch Umkreise definierte Bereiche
  3. Durch Territorien definierte Bereiche

Nach Art des Gebäudes:

  1. Custom-Funktionen
  2. Skriptnamen
  3. Entity Types
  4. Entity Categories

Vorausgesetzte Module:

  • (1) Interface Core

API.ProtectBuildingCustomFunction (_PlayerID, _Function)
Verhindert den Abriss von Gebäuden anhand der übergebenen Funktion.

Die angegebene Funktion muss eine Funktion im lokalen Skript sein. Es ist möglich Funktionen innerhalb Tables anzugeben. Die self-Referenz wird allerdings nicht unterstützt.

Eine Funktion muss true zurückgeben, wenn der Abriss geblockt werden soll. Die gleiche Funktion kann für alle Spieler benutzt werden, wenn als PlayerID -1 angegeben wird. Für welchen Spieler sie ausgeführt wird, wird stets als Parameter übergeben.

Parameters:

  • _PlayerID number ID des Spielers
  • _Function function Funktion im lokalen Skript

Returns:

    number ID der Protektion

Beispiel:

    function MyCustomProtection = function(_PlayerID, _BuildingID, _X, _Y)
   if AnythingIWant then
       return true;
   end
end
MyProtection = API.ProtectBuildingCustomFunction(1, MyCustomProtection);
